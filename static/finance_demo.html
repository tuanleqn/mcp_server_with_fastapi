üìñ How It Works

Authentication Flow:

  1. User logs in and receives a JWT token
  2. Token contains user_id, username, and roles
  3. MCP tools validate the token and extract user context
  4. Tools check if user has permission for the requested action
  5. Data is filtered to user's scope (own data only, unless admin)

Security Features:

  • üîí JWT Token Authentication: Secure token-based auth
  • üë§ User Scoping: Users can only access their own data
  • üõ°Ô∏è Role-based Access: Different permissions for user/premium/admin
  • üìù Audit Trail: All actions are logged with user context
  • ‚è∞ Token Expiration: Tokens expire for security
